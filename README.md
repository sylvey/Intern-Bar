## Backend

#### Step 0.

First, make sure that python3 is installed in your device, and complete the importation of data to database. 
確定電腦有安裝 `python3` 以及完成上述資料匯入資料庫的指令

Run the following commands:
打開終端機執行以下指令:

```shell
# for mac
cd backend
python3 -m venv venv # construct the virtual environment 建立虛擬環境 #-m: module-name
source venv/bin/activate # activate the virtual environment 啟動虛擬環境 for mac
```

```shell
# for windows
cd backend
python3 -m venv venv # construct the virtual environment 建立虛擬環境 #-m: module-name
venv\Scripts\activate.bat # activate the virtual environment 啟動虛擬環境 for windows
```

接著下載所需套件，需要的套件與版本已定義在 `requirements.txt`，下載完輸入`pip list`檢查所有用 `pip` 下載的套件。

```shell
python -m pip install --upgrade pip #pip更新至最新版本
pip install -r requirements.txt
pip list
```

#### Step 2.
Add a file named '.env' in backend folder. Paste the content below to your '.env' file.
接著可以新增名稱為 .env 的檔案於 backend資料夾，複製以下內容並將 `.env` 當中的變數改成符合電腦資料庫的值。
The name of the database is 'project', PORT = `5432`
我們該專案創建的資料庫名 NAME 為 `project`，而 PORT 為 `5432`。

```.env
SECRET_KEY={aaaaaaaaa}
DEBUG=True
ALLOWED_HOSTS=.localhost,127.0.0.1
DATABASE_URL={postgres://USER:PASSWORD@HOST:PORT/NAME}
```

#### Step 3.
Finally, sync the database and start the backend server.
If you are using pgAdmin 4 (PostgreSQL) locally as the database,
you will need to manually create the database (CREATE DATABASE) and name it project.
最後，同步資料庫並啟動 backend server。
如果是在本地端使用 `pgAdmin 4`(PostgreSQL) 作為資料庫，
會需要先自行手動操作建資料庫(CREATE DATABASE)，且命名為 `project`。 

After running python manage.py migrate,
you will notice that the previously empty DATABASE now has many TABLES,
complete with column names and various constraints and conditions such as keys.
而後執行 `python manage.py migrate`，
會發現原先空的 DATABASE 突然多了許多張 TABLE，具有欄位名稱以及鍵值等限制與條件。

```shell
python manage.py migrate
```

#### Step 4.
Once the required TABLE structure for the project is in the DATABASE,
you can proceed to import the CSV files into the database using the db_data.ipynb file in the db folder.
有了此專案所需的 TABLE 的架構於 DATABASE 中後，
接著可以透過 `db` （資料夾）中的 `db_data.ipynb` 來將其中的 csv 檔（如下）匯入資料庫。

> UserApp_user.csv </br>
> ExpApp_city.csv </br>
> ExpApp_organization.csv </br>
> ExpApp_district.csv </br>
> ExpApp_position.csv </br>
> ExpApp_experience.csv </br>
> PostApp_post.csv </br>
> PostApp_comment.csv </br>

The import order is fully listed in db_data.ipynb.
匯入的順序都有詳列在`db_data.ipynb`。  

#### Step 5.

Since data has already been imported into ExpApp and PostApp in the current database,
you need to set the starting point of the primary key sequence generated by Django automatically.
This will prevent the system from using an already existing serial primary key when adding new data later.
After opening the psql console, navigate to the database linked to this project,
and input the SQL commands generated by the previous block of instructions.
因為目前資料庫內，ExpApp 和 PostApp 已經有匯入資料，因此必須設定 django 自動生成 primary key 之序列開頭，才能避免之後新增資料時，使用已經存在的流水號 primary key。
打開 psql console 後，移到本專案所連結之資料庫，並輸入執行上個區塊指令後所產生的 sql 指令 ()。
```psql shell
\c project #剛剛匯入資料的database名稱
BEGIN;
SELECT setval(pg_get_serial_sequence('"ExpApp_experience"','exp_id'), coalesce(max("exp_id"), 1), max("exp_id") IS NOT null) FROM "ExpApp_experience";
SELECT setval(pg_get_serial_sequence('"ExpApp_position"','pos_id'), coalesce(max("pos_id"), 1), max("pos_id") IS NOT null) FROM "ExpApp_position";
SELECT setval(pg_get_serial_sequence('"ExpApp_organization"','org_id'), coalesce(max("org_id"), 1), max("org_id") IS NOT null) FROM "ExpApp_organization";
COMMIT;
BEGIN;
SELECT setval(pg_get_serial_sequence('"PostApp_post"','post_id'), coalesce(max("post_id"), 1), max("post_id") IS NOT null) FROM "PostApp_post";
SELECT setval(pg_get_serial_sequence('"PostApp_comment"','comment_id'), coalesce(max("comment_id"), 1), max("comment_id") IS NOT null) FROM "PostApp_comment";
COMMIT;
```
#### Step 6.
Afterward, complete the database synchronization and start the backend server.
隨後完成同步資料庫並啟動 backend server。
```shell
python manage.py runserver
```

Open http://127.0.0.1:8000/api/city in your browser.
If you successfully retrieve the resource, it indicates that the backend has started successfully.
用瀏覽器打開  http://127.0.0.1:8000/api/city ，如果有順利拿到資源表示後端起成功。

## Frontend

#### Step 0. Install yarn (安裝 yarn)
If you haven't installed Yarn yet, please refer to the official website: https://classic.yarnpkg.com/lang/en/docs/install/#mac-stable and follow the steps for installation.
如果你還沒有安裝 yarn，請參考官網 https://classic.yarnpkg.com/lang/en/docs/install/#mac-stable ，依照步驟安裝。

#### Step 1. Install packages (安裝 packages)
When opening the project for the first time, use the following command to install all the required packages.
You can skip this command when reopening the project in the future.
第一次開啟專案請先用以下指令安裝所有需要的 packages，之後再次開啟專案跳過這個指令即可。

```shell
cd frontend
```

```shell
yarn install
yarn
```
If yarn start results in an error, try starting from this step again to see if it resolves the issue.
如果yarn start出error的話，這一個步驟開始重新做看看。
```shell
npm install @babel/helper-compilation-targets --save-dev #如果yarn start出error的話 加這一行重新試看看
npm update #如果yarn start出error的話 加這一行重新試看看
yarn
```

#### Step 2. Start the frontend (開啟前端)
```shell
cd frontend
yarn start
```
#### Step 3. Open the browser (開啟瀏覽器 )
Normally, the browser will open automatically. If it doesn't, manually open http://127.0.0.1:3000/.
If you see the following screen, it means that the frontend has started successfully and the frontend and backend are connected.
一般情況會自動開啟瀏覽器，若沒有請手動開啟 http://127.0.0.1:3000/ ，看到以下畫面即代表成功開啟前端且前後端連接成功。
